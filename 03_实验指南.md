## 四、实验步骤

### 步骤 1：训练源模型

```bash
# 训练 CNN 模型 (QUIC22)
python train.py --config configs/train_quic22_cnn.yaml

# 预期输出:
#   Epoch 50/50 | Train Acc: 0.92xx | Val Acc: 0.90xx | Val F1: 0.88xx
#   Test Accuracy: 0.90xx
#   Training complete. Results saved to outputs/quic22_cnn/
```

**产出文件**：
- `outputs/quic22_cnn/best_model.pt` — 最佳模型权重
- `outputs/quic22_cnn/baseline_entropy.npy` — 基线熵（用于漂移检测）
- `outputs/quic22_cnn/train_results.json` — 训练结果

**训练时间估算** (RTX 3080, size=S):
- CNN: ~2-5 分钟/epoch × 50 epochs = ~2-4 小时
- Transformer: ~5-10 分钟/epoch × 50 epochs = ~4-8 小时

### 步骤 2：短期评估（所有方法 on QUIC22 W-45）

```bash
python evaluate_tta.py \
    --config configs/eval_quic22.yaml \
    --checkpoint outputs/quic22_cnn/best_model.pt \
    --output-dir outputs/eval_quic22_single \
    --mode single
```

**预期输出**：

```
=== B1: Static (no adaptation) ===
Accuracy: 0.78xx, F1: 0.72xx          ← 应该看到约 13% 的下降

=== B4: Tent ===
Accuracy: 0.80xx, F1: 0.74xx

=== B10: TTA-TC (Ours) ===
Accuracy: 0.87xx, F1: 0.83xx          ← 目标：显著优于 Static 和通用 TTA
```

**关键验证**：Static 结果应复现 ~13% 准确率下降（与 TMA 2023 一致）。

### 步骤 3：连续评估（QUIC22 三周）

```bash
python evaluate_tta.py \
    --config configs/eval_quic22.yaml \
    --checkpoint outputs/quic22_cnn/best_model.pt \
    --output-dir outputs/eval_quic22_sequential \
    --mode sequential
```

**关键看点**：
- Static 的 ARR 逐周下降
- 通用 TTA（Tent/EATA）是否在 W-47 出现误差累积
- TTA-TC 是否保持 ARR > 0.95

### 步骤 4：消融实验

```bash
# A1: SSL 任务选择
python run_ablation.py \
    --ablation-config configs/ablation_ssl_tasks.yaml \
    --checkpoint outputs/quic22_cnn/best_model.pt \
    --output-dir outputs/ablations/ssl_tasks

    --ablation-config configs/ablation_mask_ratio.yaml \

    --output-dir outputs/ablations/mask_ratio

    --ablation-config configs/ablation_adapt_depth.yaml \
    --checkpoint outputs/quic22_cnn/best_model.pt \
    --output-dir outputs/ablations/adapt_depth

    --checkpoint outputs/quic22_cnn/best_model.pt \
```

## 八、常见问题

### Q: cesnet-datazoo 下载失败

手动从 Zenodo 下载：
- QUIC22: 搜索 "CESNET-QUIC22" on zenodo.org
- TLS22: 搜索 "CESNET-TLS-Year22" on zenodo.org
将 HDF5 文件放入 `./data/` 目录。

### Q: 显存不足 (OOM)

减小 batch_size：修改 `configs/train_quic22_cnn.yaml` 中的 `batch_size: 256` → `128` 或 `64`。

### Q: 训练速度慢

- 确认 GPU 在使用：运行时检查 `nvidia-smi`
- 减少 `num_workers`（Windows 上可能需设为 0）
- 使用 size="XS"（更小的数据子集）进行快速调试

### Q: Windows 上 num_workers > 0 报错

在配置文件中设置 `num_workers: 0`。Windows 的多进程数据加载有兼容性问题。

### Q: 如何只跑某个方法

修改 `configs/eval_quic22.yaml` 中的 `methods` 列表：

```yaml
methods:
  - static
  - tta_tc    # 只评估 static 和 TTA-TC
```
