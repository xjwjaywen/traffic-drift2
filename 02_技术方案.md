# TTA-TC 技术设计文档

## 一、问题形式化

**设定**：分类器 $f_\theta: \mathcal{X} \to \mathcal{Y}$ 在时间段 $T_0$ 的流量上训练，部署后在 $T_1, T_2, \ldots$ 分类流量。由于概念漂移：$P_{T_0}(X, Y) \neq P_{T_k}(X, Y)$。

**目标**：仅使用 $T_k$ 的无标注流量适应 $f_\theta$，不访问 $T_0$ 的训练数据或 $T_k$ 的标签。

**输入表示**：每条流 $x$ 由前 $N=30$ 个包的三元组表示（CESNET PPI 格式）：
- 包大小：$\mathbf{s} = (s_1, \ldots, s_{30}) \in \mathbb{R}^{30}$
- 方向：$\mathbf{d} = (d_1, \ldots, d_{30}) \in \{-1, +1\}^{30}$
- 包间时间：$\mathbf{t} = (t_1, \ldots, t_{30}) \in \mathbb{R}_{\geq 0}^{30}$

合并为张量 $\mathbf{x} \in \mathbb{R}^{3 \times 30}$。

## 二、模型架构

### 2.1 Y 形组合模型 (TTATCModel)

```python
# 文件: tta_tc/models/tta_tc_model.py

class TTATCModel(nn.Module):
    encoder    # 共享编码器 (CNN 或 Transformer)
    cls_head   # 分类头 (FC → C 类)
    ssl_head   # SSL 头 (MPFP/POP/FSR 子头)
```

**前向流程**：
1. `h = encoder(ppi, flow_stats)` → 隐层表示 `(B, hidden_dim)`
2. `logits = cls_head(h)` → 分类 logits `(B, C)`
3. SSL 头通过 `ssl_head.forward_mpfp/pop/fsr` 分别计算各任务损失

**适应参数选择**：
- `"norm_only"`：仅更新 GroupNorm/LayerNorm 参数（最保守）
- `"encoder"`：更新整个编码器（默认）
- `"ssl_head"`：同时更新编码器 + SSL 头

### 2.2 编码器 A：1D-CNN

```python
# 文件: tta_tc/models/cnn_encoder.py

输入: (B, 3, 30)
  → Conv1d(3, 64, k=5, p=2) → GroupNorm(8, 64) → ReLU
  → Conv1d(64, 128, k=5, p=2) → GroupNorm(8, 128) → ReLU
  → Conv1d(128, 256, k=3, p=1) → GroupNorm(8, 256) → ReLU
  → GlobalAvgPool → (B, 256)
  → 可选 flow_stats 拼接 → FC 融合
输出: (B, hidden_dim)
```

- 参数量：~200K
- 使用 GroupNorm (8 组) 替代 BN

### 2.3 编码器 B：Transformer

```python
# 文件: tta_tc/models/transformer_encoder.py

输入: (B, 3, 30)
  → 线性投影每个包位置: (B, 30, d_model)
  → 加位置编码 + [CLS] token
  → TransformerEncoderLayer × L 层 (LayerNorm, Multi-Head Attn)
  → [CLS] token 输出 → (B, d_model)
输出: (B, hidden_dim)
```

- 默认配置：L=4 层, d_model=256, nhead=4
- 使用 LayerNorm（PyTorch Transformer 默认）
- `return_all_tokens=True` 时返回所有 token 表示（供 MPFP 使用）

### 2.4 SSL 头

```python
# 文件: tta_tc/models/heads.py

class SSLHead(nn.Module):
    # MPFP 子头：
    mpfp_proj → FC(h, h) → ReLU
    mpfp_size → FC(h, 1)      # 预测包大小 (回归)
    mpfp_dir  → FC(h, 1)      # 预测方向 (二分类)
    mpfp_ipt  → FC(h, 8)      # 预测 IPT bin (8 类)

    # POP 子头：
    pop_head → FC(h, h/2) → ReLU → FC(h/2, 6)  # 6 种排列

    # FSR 子头：
    fsr_head → FC(h, h/2) → ReLU → FC(h/2, 6)  # 6 维统计量
```

## 三、自监督任务详细设计

### 3.1 MPFP: Masked Packet Feature Prediction

**文件**: `tta_tc/ssl_tasks/mpfp.py`

**机制**：
1. 随机选择 15% 的包位置（约 4-5 个位置）
2. 将被选位置的 SIZE/DIR/IPT 三个特征置零
3. 编码器处理掩码后的输入
4. SSL 头预测被掩码位置的原始值

**损失函数**：
```
L_MPFP = (1/|M|) Σ_{i∈M} [MSE(ŝ_i, s_i) + BCE(d̂_i, d_i) + CE(t̂_i, bin(t_i))]
```

**IPT 离散化**：8 个对数 bin（匹配 CESNET PHIST 格式）

| Bin | 范围 |
|-----|------|
| 0 | 0–15 ms |
| 1 | 16–31 ms |
| 2 | 32–63 ms |
| 3 | 64–127 ms |
| 4 | 128–255 ms |
| 5 | 256–511 ms |
| 6 | 512–1023 ms |
| 7 | ≥1024 ms |

**为什么有效**：当流量分布漂移时（如 Google 证书更换改变位置 5 的包大小），重建误差增大，提供梯度信号重新校准编码器表示。

### 3.2 POP: Packet Order Prediction

**文件**: `tta_tc/ssl_tasks/pop.py`

**机制**：
1. 将 30 个包位置分为 3 段：[0:10], [10:20], [20:30]
2. 随机排列 3 段（共 6 种排列）
3. 编码器处理打乱后的 PPI
4. 从 [CLS] 表示预测原始排列索引

**损失**：6 分类交叉熵

**为什么互补 MPFP**：POP 捕捉时序流结构（请求-响应模式），与 MPFP 的逐位置特征统计正交。

### 3.3 FSR: Flow Statistics Reconstruction

**文件**: `tta_tc/ssl_tasks/fsr.py`

**机制**：
1. 从 PPI 序列计算 6 个聚合统计量作为目标：
   - 平均包大小、最大包大小、包大小标准差
   - 上行包占比
   - 平均 IPT、IPT 标准差
2. 编码器处理原始 PPI，从 [CLS] 表示预测这些统计量

**损失**：MSE 回归损失

**为什么有用**：迫使编码器学习微观（逐包）与宏观（流级）特征的关系，该关系在应用更新时可能改变。

### 3.4 组合 SSL 损失

```python
# 文件: tta_tc/ssl_tasks/combined.py

L_ssl = L_MPFP + α · L_POP + β · L_FSR
# 默认: α = 0.2, β = 0.1 (MPFP 提供最密集的梯度信号，主导损失)
```

## 四、适应引擎

### 4.1 7 步流水线

**文件**: `tta_tc/tta/tta_engine.py`

```
对每个到达的测试批 B:

Step 1: OOD 过滤
  ├── 计算能量分数 E(x) = -log Σ_c exp(z_c)
  └── 排除 E(x) > E_threshold 的流（可能是未知应用）

Step 2: 熵过滤 (SAR 风格)
  ├── 计算预测熵 H(x) = -Σ p_c log p_c
  └── 排除 H(x) > 0.4 · ln(C) 的流（不可靠样本）

Step 3: PBRS 缓冲
  ├── 将有效样本加入预测平衡采样缓冲区
  └── 从缓冲区采样类别平衡的 mini-batch

Step 4: 类加权 SSL 损失
  ├── 在采样 mini-batch 上计算 L_ssl
  └── 按类别逆频率加权

Step 5: 选择性更新
  ├── 检测漂移类（ΔH_c > τ_H）
  ├── 突变漂移: 高学习率 × γ，多步适应
  └── 渐变漂移: 正常学习率，Fisher 约束

Step 6: 抗遗忘
  ├── Fisher 正则化: L += λ Σ F_i(θ_i - θ_i^source)²
  └── 随机恢复: 每个参数以 p=0.01 恢复为源值

Step 7: EMA 教师更新
  └── θ_teacher ← m · θ_teacher + (1-m) · θ_e, m=0.999
```

### 4.2 漂移检测器

**文件**: `tta_tc/tta/drift_detector.py`

- 维护每类平均熵的滑动窗口
- 对比当前熵与基线熵（训练数据计算）
- 检测两种漂移：
  - **突变漂移**：`dH/dt > τ_abrupt`（短时间内熵剧增）→ 禁用随机恢复，高学习率
  - **渐变漂移**：`ΔH > τ_H`（熵缓慢上升）→ 正常适应

### 4.3 PBRS 缓冲区

**文件**: `tta_tc/tta/pbrs_buffer.py`

- 固定大小的 reservoir buffer（默认 10 × C）
- 按预测类别平衡采样
- 解决流量突发性：同一应用大量流不会主导缓冲区
- 采样时返回类别逆频率权重

### 4.4 抗遗忘机制

**文件**: `tta_tc/tta/anti_forgetting.py`

**Fisher 正则化**：
```
L_fisher = λ · Σ_i F_i · (θ_i - θ_i^source)²
```
- F_i 通过源数据计算（训练结束后一次性计算并保存）
- 限制重要参数的偏移，允许不重要参数自由适应

**随机恢复**：
- 每步以概率 p=0.01 将每个参数恢复为源值
- 持续注入源知识，防止长期遗忘
- 突变漂移时暂停（避免抵消有效适应）

## 五、基线实现

| 基线 | 文件 | 核心实现 |
|------|------|----------|
| BN-Adapt | `baselines/bn_adapt.py` | BN 层设为 train 模式，自动更新运行统计量 |
| Tent | `baselines/tent.py` | 在 Norm 参数上最小化预测熵 |
| EATA | `baselines/eata.py` | 熵过滤 + Fisher 正则化 + 冗余过滤 |
| CoTTA | `baselines/cotta.py` | 增强平均伪标签 + 随机恢复 + EMA 教师 |
| SAR | `baselines/sar.py` | 熵过滤 + 锐度感知最小化 (SAM) |
| NOTE | `baselines/note.py` | 实例感知 BN + PBRS 缓冲 + 熵最小化 |

所有基线统一接口：`adapt_batch(ppi, flow_stats) → (logits, info)`

## 七、训练流程

**文件**: `train.py`

```
输入: configs/train_quic22_cnn.yaml

1. 加载数据 (CESNET DataZoo)
2. 创建 TTATCModel (编码器 + 分类头 + SSL 头)
3. 每个 epoch:
   a. 训练: L = L_cls + λ · L_ssl
   b. 验证: Accuracy, Macro-F1
   c. 保存最佳模型 (按 Val F1)
4. 测试集评估
5. 保存 baseline_entropy.npy (用于漂移检测)
6. 保存 train_results.json
```

**联合训练的关键**：SSL 损失在训练阶段就参与优化，确保编码器对 SSL 任务和分类任务同时有效。这是 TTT 框架的核心设计要求。

## 八、评估流程

**文件**: `evaluate_tta.py`

### 8.1 单期评估 (mode=single)

在单个测试期（如 W-45）上评估所有方法，输出：
- 每个方法的 Accuracy, Macro-F1, 适应时间

### 8.2 连续评估 (mode=sequential)

模型在多个测试期上连续适应（不重置），输出：
- 每期 Accuracy, Macro-F1, ARR
- 整体 AURC

**连续评估的关键**：TTA 方法在期间切换时不重置模型权重，模拟真实部署的持续适应。
