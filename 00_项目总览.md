# TTA-TC 项目总览

## 一、研究问题

加密流量分类器在部署后性能随时间快速衰减（概念漂移）。Luxemburk & Cejka (TMA 2023) 在 CESNET-QUIC22 数据集上发现，模型训练一周后准确率下降 13.48%。现有工作仅报告了这一现象，未提出有效缓解方案。

**核心挑战**：
- 流量分布因应用更新、证书轮换、协议演进而持续变化
- 标注新数据成本极高（需要 DPI 或人工标注）
- 定期重训不可行（需频繁重训+大量标注数据）

## 二、研究思路

**核心 Idea**：将测试时自适应（Test-Time Adaptation, TTA）技术应用于加密流量分类，通过流量特定的自监督辅助任务在部署时无标注适应模型。

**为什么可行**：
1. TTA 在计算机视觉领域已成熟（Tent, EATA, CoTTA, SAR, NOTE 等 20+ 方法），但**从未**应用于流量分类
2. 流量数据具有天然的自监督信号（包大小序列、方向模式、时序结构），可设计流量特定的辅助任务
3. CESNET 公开数据集提供真实的时间漂移评估基准

**关键创新点**：
1. **首次将 TTA 应用于流量分类**——填补 CV/NLP 与网络安全的跨领域空白
2. **流量特定的 SSL 辅助任务**——MPFP/POP/FSR 三任务设计
3. **类感知选择性适应**——针对流量漂移的非均匀性
4. **年尺度时间评估**——首次在 CESNET-TLS-Year22 上评估漂移缓解

## 三、方法概要

### 3.1 架构：Y 形模型

```
输入流 x = (包大小, 方向, IPT)
         │
    ┌─────┴──────┐
    │  共享编码器  │  ← 测试时更新
    │  (CNN/Trans) │
    └──┬─────┬───┘
       │     │
  ┌────┴──┐ ┌┴────────┐
  │ 分类头 │ │  SSL头   │  ← 测试时更新
  │ (冻结) │ │(MPFP+POP │
  └───┬───┘ │ +FSR)    │
      │     └────┬────┘
   预测ŷ     L_ssl (无标注损失)
```

### 3.2 三个自监督任务

| 任务 | 机制 | 损失 | 直觉 |
|------|------|------|------|
| **MPFP** (主) | 掩码 15% PPI 位置，预测包大小/方向/IPT | MSE + BCE + CE | 当流量分布漂移时，重建误差增大→提供梯度信号 |
| **POP** (辅) | 打乱 3 段 PPI，预测原始排列 (6 类) | CE | 捕捉时序交互模式的漂移 |
| **FSR** (辅) | 从 PPI 预测流级统计量 | MSE | 学习微观-宏观特征关系的变化 |

### 3.3 7 步适应流水线

1. **OOD 过滤**：能量分数排除未知应用
2. **熵过滤**：排除高不确定性样本 (SAR 风格)
3. **PBRS 缓冲**：预测平衡采样，解决流量突发性
4. **类加权 SSL 损失**：逆频率加权，防止大类主导
5. **选择性更新**：漂移类用高学习率，稳定类用 Fisher 约束
6. **抗遗忘**：Fisher 正则化 + 随机参数恢复
7. **EMA 教师更新**：稳定伪标签生成

### 3.4 关键设计决策

- **GroupNorm/LayerNorm** 替代 BatchNorm：BN 是 TTA 不稳定的首要来源（流量突发导致批统计量偏差）
- **冻结分类头**：只通过 SSL 损失更新编码器，避免直接修改分类边界
- **漂移类型感知**：突变漂移（证书轮换）用高学习率+多步适应；渐变漂移（版本更新）用低学习率+正则化

## 四、实验设计

### 4.1 数据集

| 数据集 | 协议 | 流量数 | 类别数 | 时间跨度 | 用途 |
|--------|------|--------|--------|----------|------|
| CESNET-QUIC22 | QUIC | 153M | 102 | 4 周 | 短期漂移评估 |
| CESNET-TLS-Year22 | TLS | 507.7M | 180 | 12 个月 | 长期漂移评估 |

### 4.2 时间协议

- **QUIC22**：W-44 训练 → W-45/46/47 测试（含 Google 证书轮换事件）
- **TLS-Year22**：M1-3 训练 → M4-12 连续测试（9 个月）

### 4.3 对比方法 (10 个)

| 方法 | 类型 | 出处 |
|------|------|------|
| Static | 无适应 | - |
| BN-Adapt | 统计更新 | Schneider 2020 |
| Tent | 熵最小化 | ICLR 2021 |
| EATA | 高效 TTA | ICML 2022 |
| CoTTA | 持续 TTA | CVPR 2022 |
| SAR | 鲁棒 TTA | ICLR 2023 |
| NOTE | 非 i.i.d. TTA | NeurIPS 2022 |
| Self-evolving | 流量特定 | arXiv 2025 |
| **TTA-TC (Ours)** | **流量 SSL TTA** | **本工作** |

### 4.4 评估指标

- **Accuracy / Macro-F1**：标准分类指标
- **ARR** (Accuracy Retention Rate)：`Acc(t) / Acc(t=0)`，衡量准确率保持程度
- **AURC** (Area Under Retention Curve)：ARR 的时间积分，衡量整体时间鲁棒性
- **Per-Class Recall Decay**：每类召回率衰减，分析非均匀漂移
- **适应延迟**：每批适应的挂钟时间

### 4.5 消融实验 (10 组)

| 编号 | 消融 | 验证目标 |
|------|------|----------|
| A1 | SSL 任务选择 | MPFP vs POP vs FSR vs 组合 |
| A2 | 掩码比例 | 5%/10%/15%/25%/50% |
| A3 | 适应深度 | Norm 层 vs 完整编码器 |
| A4 | 类感知 vs 全局 | 选择性适应的价值 |
| A5 | 抗遗忘 | Fisher/随机恢复/组合 |
| A6 | 批大小敏感性 | 16/32/64/128/256 |
| A7 | 骨干网络 | CNN vs Transformer |
| A8 | 归一化类型 | BN vs GN vs LN |
| A9 | 适应频率 | 每批/10批/100批 |
| A10 | 梯度对齐 | 验证 SSL 与 CLS 梯度相关性 |

## 五、核心假设

- **H1**：TTA-TC + MPFP 的 ARR 优于所有通用 TTA 方法（B4-B8），因为流量特定 SSL 提供更好对齐的梯度
- **H2**：类感知选择性适应优于全局适应，因为漂移是非均匀的
- **H3**：GroupNorm/LayerNorm 对流量 TTA 至关重要（BN 在突发流量下不稳定）
- **H4**：MPFP + POP 组合优于单任务（捕捉特征级+结构级漂移）
- **H5**：TTA-TC 在 QUIC22 上 3 周保持 ARR > 0.95，在 TLS-Year22 上 6 个月保持 ARR > 0.90

## 八、风险矩阵

| 风险 | 可能性 | 影响 | 缓解 |
|------|--------|------|------|
| SSL 梯度与分类不对齐 | 中 | 高 | 多任务消融；显式测量对齐；退回熵 TTA |
| 长期误差累积 | 中 | 高 | Fisher 正则 + 随机恢复 + 定期重置 |
| GN/LN 降低基线精度 | 低 | 中 | 训练时对比 BN vs GN/LN |
| 类不平衡破坏适应 | 中 | 中 | PBRS + 类加权损失 |
| 计算开销过大 | 低 | 中 | 仅适应 Norm 参数 + SSL 头 |
